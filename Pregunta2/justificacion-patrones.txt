Patron Factory Method:
En el codigo original, en la función main, se tenia una sentencia condicional if, la cual tenia como objetivo crear una instancia de una de las dos clases de cine a partir de un parametro (numero) que era solicitado por el sistema. Para poder aplicar el patron Factory Method, se ha creado una clase externa llamada CineFactory, la cual tendra una función que recibira como parametro el numero del cine, y donde se centralizara la creación de las instancias de los cines mediante la sentencia condicional if antes mencionada. Con lo cual esta sera retirada de la funcion main y sera puesta en la funcion obtener_cines de la clase CineFactory.

Patron Singleton:
Partiendo de la modificación hecha anteriormente para el patron Factory, se tiene la función obtener_cines retorna una instancia ya sea de CineStark o de CinePlaneta, esto se realiza cada vez que el usuario selecciona el cine de su preferencia, por lo cual, cada vez que se pase por este metodo se creara una instancia nueva, ocupando un espacio en la memoria distinto. Para este caso se podria usar el patron Singleton para que cada vez que se instancie alguna de esas clases, apunten al mismo espacio de memoria. En este caso se crea un metodo de clase llamado get_instance el cual retornara una instancia de la clase, si es que esta no existe.

Patron Composite:
Para este caso, en el codigo original, la función main imprimia la lista de los cines mediante un codigo "hard-codeado", cuyo problema era que si querias agregar un nuevo cine, tenias que ir a todos los print que habian para agregar el nuevo cine. Para aplicar el patron Composite en este caso, se creo una clase "padre" llamada Cine la cual definia los metodos de los dos cines y además se aumento los metodos add_hijos y imprimir_cine, el primero usado para agregar cada cine existente a una lista y el otro que serve para recorrer esa lista y listar los nombres de los cines. Cabe resaltar que para que esto funcione, tanto CinePlaneta como Cine Stark deben heredar de Cine. Con esto ya no es necesario estar añadiendo el print de cada nuevo cine que se cree.

Patron Decorator:
Para este ultimo patron, se toma el bucle for definido en el codigo original, el cual tiene como funcionalidad listar las peliculas de un cine previamente escogido. Para este caso el Patron Decorator se utilizar para "decorar" esta parte del codigo. Lo primero que se hace es crear dos funciones, una que contendra el bucle, la cual lleva por nombre mostrar_pelicula, esta recibe como parametro la lista de peliculas. La segunda funcion es la denominada función decoradora que recibira como parametro una función. Si bien es cierto el patron decorator sirve principalmente para añadir funcionalidad a un metodo sin tocar el codigo de este, se puede utilizar tambien para este tipo de casos (En nuestra opinion).
